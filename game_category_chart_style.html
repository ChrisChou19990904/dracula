二、後端存在的原因

後端的存在是為了確保遊戲的「狀態主控權不在玩家手上」，
而不是單純因為「遊戲世界大」或「進度多」。

三、資料庫選擇概述

  • SQL（MySQL / MariaDB / PostgreSQL / Oracle）：結構化資料、課金、排行榜
  • NoSQL（MongoDB / Redis / Cassandra / DynamoDB）：高並發、即時遊戲狀態、分散式需求
  • 混合架構：核心資料用 SQL，快取 / 排行 / session 用 Redis，物品/地圖/任務可能用自定義儲存

四、判斷是否需要後端的三個問題

  1. 玩家能否修改存檔？
  2. 玩家之間會互相影響嗎？
  3. 公司是否需控制經濟 / 排行 / 防作弊？

如果全部否 → 不需要後端；任一是 → 一定要後端
